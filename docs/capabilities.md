## How to Add or Update a Capability

- Export a static `describe()` method or object from your agent, plugin, adapter, or utility.
- Ensure your describe output includes: name, description, usage, and (if applicable) methods, events, schemas, and references.
- Run the describe registry aggregation script (or CI will do this automatically).
- Add or update a markdown file in `docs/capabilities/` for your capability.
- The registry and this index will update automatically.

## 🧠 ContextManager (Advanced Context & Memory Management)

- **Manifest-driven context tiering:** All context/cache files are indexed in a manifest with metadata (tier, lastAccessed, size, etc.).
- **Policy-based migration:** Files are automatically migrated between hot/warm/cold tiers based on recency and size.
- **CLI commands:**
  - `ai-helpers context-enforce-tiering` — Enforce manifest-driven tiering policies.
  - `ai-helpers context-migrate-tier <fileName> <targetTier>` — Migrate a file to a different tier.
- **HTTP API endpoints:**
  - `POST /cache/enforce-tiering` — Enforce tiering policies.
  - `POST /cache/migrate-tier` — Migrate a file to a different tier (body: `{ fileName, targetTier }`).
- **Auto-documented:** This feature is always up to date via the describe registry and is surfaced in this canonical index.
